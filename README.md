# 🗄️ 디비디비-딥 (DBDB-Deep)

> **"데이터베이스의 깊은 곳까지 내려가는 딥다이브 스터디"**  
> 우아한테크코스 8기 크루들이 모여 각자의 DB 역량을 강화하기 위해 운영되는 스터디입니다.

---

## 🎯 스터디 목표

- **DB 실력 키우기**: DB의 내부 동작 원리, 쿼리 최적화 등 데이터베이스에 대해 깊이 있게 스스로 공부합니다.
- **꾸준히 하기**: 학습 목표를 세우고, 틈틈이 공부하여 목표를 달성합니다.
- **함께 성장하기**: 매주 세미나를 통해 공부한 내용을 공유하고 피드백하며 지식을 나눕니다.

---

## ⚙️ 운영 방식

1. 각자 정한 도서나 강의를 2주 동안 공부하고 기록합니다.
2. 매주 정해진 시간에 모여서 그룹별로 세미나를 진행합니다.
3. 발표 시간은 20~30분 정도로, 자유로운 형식으로 진행합니다.
4. 발표 후에는 자유롭게 질문하고 의견을 주고받으며 피드백을 나눕니다.
5. 모든 멤버가 본인의 목표를 달성할 때까지 스터디를 지속합니다.

---

## 📘 운영 규칙

1. **참석**: 세미나에는 꼭 참석하도록 하며, 부득이하게 빠질 경우 미리 팀원들에게 알린다.
2. **자료 공유**: 발표 자료는 세미나 시작 전까지 GitHub에 업로드한다.
3. **질문**: 발표를 듣고 궁금한 점이나 의견을 적극적으로 나누며, 발표마다 1개 이상의 질문을 권장한다.
4. **주제 선택**: 딥다이브 주제 or 발표 주제가 다른 사람을 의식하지 않고 자유롭게 정한다.
5. **학습 내용**: 단순 요약보다는 공식 문서나 직접 테스트해 본 결과를 바탕으로 깊이 있게 공부한다.
6. **기록**: 세미나에서 논의된 좋은 내용이나 답변은 나중에도 볼 수 있도록 자료에 함께 기록해 둔다.

---

## 🛠 참여 방법 (GitHub 활용)

1. **브랜치 생성**: `git checkout -b [이름]`으로 본인 브랜치를 생성합니다.
2. **개인 폴더**: 루트 디렉토리에 본인의 닉네임으로 폴더를 만듭니다. (예: `lochi/`)
3. **기록**: 공부한 내용과 발표한 내용은 개인 폴더 안에 자유롭게 업로드합니다. (필수)
4. **공유**: 작성이 끝나면 `main` 브랜치로 Pull Request를 보냅니다. (세미나 전까지 완료 권장)
